# 决策项目结构与回填规则

这份文档给 `dbs-decision` 提供具体操作规则。

## 一、项目根目录

每个项目固定使用：

```text
~/.dbs/decisions/{项目名}/
```

如果用户加 `--here` 或明确要求"放在当前项目里"，落到：

```text
{当前工作目录}/决策/{项目名}/
```

根目录下固定保留：

- `00_使用说明.md`
- `AGENTS.md`
- `SOURCE_OF_TRUTH.md`
- `我的当前状态.md`
- `01_事实/`
- `02_规律/`
- `03_定格/`
- `04_待解/`
- `99_归档/`

## 二、四层目录建议文件

### `01_事实/`

- `_这层放什么.md`
- `客观事实.md`
- `用户长期偏好.md`
- `相关方笔记/`（隐私模式下文件名用代号）
- `相关方笔记/代号映射.md`（仅隐私模式，不分享、不提交）

业务项目常见追加：

- `产品事实.md`
- `价格带与产品矩阵.md`
- `合作方与角色.md`
- `场域约束.md`
- `已验证结果.md`

### `02_规律/`

- `_这层放什么.md`
- `规律索引.md`
- `概念_01_*.md`
- `模式_01_*.md`

### `03_定格/`

- `_这层放什么.md`
- `状态画像_YYYY-MM.md`：月度横切
- `分析_YYYY-MM-DD_主题.md`：事件后的诊断
- `对象排序_YYYY-MM-DD.md` 或 `决策地图_YYYY-MM.md`：专项

### `04_待解/`

- `_这层放什么.md`
- `开放问题.md`
- `待验证假设.md`
- `观察指标.md`
- `决策事件_YYYY-MM-DD_标题.md`（关键分叉时新建）

## 三、`_这层放什么.md` 写什么

每个层目录第一行讲清楚两件事：

1. 这层收什么
2. 这层不收什么

例子：

```markdown
# 01_事实 放什么

收：客观发生过的事、相关方的稳定信息、用户自己的长期偏好和边界、已确认的事实。

不收：你的判断、你的推测、阶段性结论、还没验证完的假设。这些去 02_规律 或 04_待解。

规则：只追加，不改写已有条目。事实和判断分开写。
```

## 四、默认工作流

### Step 1：先更新 `我的当前状态.md`

默认不是先立案，是先更新当前工作态。

固定包含：

```markdown
# 我的当前状态

更新时间：YYYY-MM-DD

## 当前阶段
## 当前最强矛盾
## 当前主线
## 最近节点
## 待回填结果
## 暂不处理
## 当前判断
## 最近动向
## 下次对话推荐入口
```

### Step 2：判断新信息属于哪一层

| 新信息 | 去哪里 |
|---|---|
| 稳定事实、相关方近况 | `01_事实/` |
| 反复出现的模式 | `02_规律/`（可先用暂定标签） |
| 还没验证的假设、要观察的指标 | `04_待解/` |
| 重大分叉、后续必须回填 | `04_待解/决策事件_*.md` |

### Step 3：只有重大分叉才建事件文件

触发立案的 5 种情况：

1. 涉及产品结构变化
2. 涉及重大合作变化
3. 涉及价格带变化
4. 涉及长期策略反转
5. 后续明确需要结果回填

## 五、更新当前状态时至少改什么

1. `我的当前状态.md`
2. 必要时 `01_事实/` 中对应文件
3. 必要时 `04_待解/待验证假设.md`
4. 必要时 `04_待解/观察指标.md`
5. 只有遇到关键分叉时进入 `04_待解/决策事件_*.md`

## 六、立案时至少改什么

1. 新建 `04_待解/决策事件_YYYY-MM-DD_标题.md`
2. `04_待解/开放问题.md` 追加索引行
3. `我的当前状态.md` 写明这个分叉当前在哪
4. 必要时补 `待验证假设.md`、`观察指标.md`

## 七、回填时至少改什么

1. 对应 `决策事件_*.md` 的 § 8 追加 `[结果回填 YYYY-MM-DD]`
2. `01_事实/客观事实.md`（写已确认事实）
3. `我的当前状态.md`
4. `04_待解/` 里对应的开放问题或假设——清掉或标"已完成"

如果验证结果改变了某个概念或模式：

5. `02_规律/` 对应文件追加 `[修正 YYYY-MM-DD]`，不重写

## 八、哪些文件能改、哪些不能改

### 持续覆盖更新

- `我的当前状态.md`

### 只追加，不改已有条目

- `01_事实/*`
- `04_待解/开放问题.md`、`待验证假设.md`、`观察指标.md` 完成后允许移除条目，但不允许重写

### 缓慢追加修正，原段不重写

- `02_规律/概念_*.md`
- `02_规律/模式_*.md`
- `02_规律/规律索引.md`

修正方式：在原段下方追加 `[修正 YYYY-MM-DD]` 段落。

### 写完不改，新版本另起

- `03_定格/*` 全部

### 只读

- `99_归档/*`

## 九、决策事件模板

```markdown
# 决策事件：{标题}

日期：YYYY-MM-DD
状态：进行中 / 待回填 / 已完成

## 1. 决策背景
## 2. 原始输入
## 3. 已确认的事实
## 4. 用户的工作性判断
## 5. AI 的工作性判断
## 6. 当前执行计划
## 7. 观察指标
## 8. 结果回填
## 9. 教训 / 后续影响
## 10. 关联文件
```

## 十、快照模板

每份 `03_定格/` 文件至少包含：

1. 时间和范围
2. 当前客观处境
3. 当前主线
4. 最稳定的理解
5. 还没解决的张力
6. 下一阶段最该盯的变量

## 十一、`SOURCE_OF_TRUTH.md` 起什么作用

解决两个问题：

1. 想找某类信息时，知道去哪里
2. 同一信息多个版本冲突时，谁优先

至少写出：

- 当前状态看哪里：`我的当前状态.md`
- 稳定事实看哪里：`01_事实/`
- 规律和模式看哪里：`02_规律/`
- 最新快照看哪里：`03_定格/` 下日期最新的
- 待解事项看哪里：`04_待解/`
- 关键决策看哪里：`04_待解/决策事件_*.md`

冲突时本地文件优先。AI 对话记忆只是临时缓存。

## 十二、`AGENTS.md` 起什么作用

写给 AI 看的协作规则。每个新建项目自动生成，至少包含：

1. 项目定位（"这是一个 X 领域的决策跟踪项目"）
2. 进入对话先读哪些文件
3. 来源标签清单和铁律
4. 四层规则速查表
5. 用户多条信息时的综合画像模式
6. 何时不该继续提供方案（"问什么办 → 拒方案 → 问新的"循环）
7. 对话结束时该做什么
8. （隐私模式）代号纪律

模板见 `decision_AI协作规则.md`。
