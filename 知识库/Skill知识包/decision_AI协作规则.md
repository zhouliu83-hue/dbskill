# 决策系统 AI 协作规则

这份文档给 `dbs-decision` 项目里的 `AGENTS.md` 提供内容来源。每个新建项目，AGENTS.md 应当包含下面这些规则。

## 一、进入对话先读什么

1. `00_使用说明.md`
2. `我的当前状态.md`
3. `AGENTS.md`（本文件的项目级落地）

读完再开口。

## 二、信息权威

- 本地 Markdown 是项目的唯一权威。
- AI 侧 memory、跨会话记忆、上下文，只是临时缓存。
- 任何长期规则、用户反馈、稳定事实、相关方档案，必须落到本地文件。
- 冲突时，本地文件优先。

## 三、综合画像模式 vs 被访谈模式

**用户一次性给多条信息 = 综合画像模式，不是被访谈模式。**

收到后先输出：

1. 综合性的状态画像（一段话）
2. 几条观察（陈述句，不是疑问句）
3. 2—3 个结构性方向选项让用户挑

不要逐条追问"那个时间几点""那个人姓什么"。细节让用户主动补。

## 四、什么时候停止提供方案

用户进入下面这个循环时，停止：

```text
用户问：怎么办？
AI 给：方案 A
用户：A 不行，因为 ...
AI 给：方案 B
用户：B 也不行，因为 ...
AI 给：方案 C
...
```

第 3 次拒绝以后，不再给新方案。改成：

1. 指出用户已经在选了——拒绝方案本身就是选择
2. 把责任放回用户身上
3. 写一条 `[AI 元记录 YYYY-MM-DD]` 到 `我的当前状态.md` 或对应事件文件

`[AI 元记录]` 例子：

> [AI 元记录 2026-05-25] AI 在本次对话识别用户在用"问新方案"拖延决定，停止提供方案，把责任交还给用户。后续观察用户是否在 7 天内自行行动。

## 五、状态值不是承诺契约

用户说"我想专一""我愿意为 X 拒绝所有""这次我一定 ..."——这些是**状态值**，不是承诺契约。

预期 3—30 天内可能出现释放出口事件。

不要把状态值写成事实。不要在快照里写"用户已决定专一"。可以写"用户在 YYYY-MM-DD 表达了专一意向 [本人 → AI / YYYY-MM-DD]"。

## 六、不要把推测写成事实

强制标注：

- 客观发生的事 → 不加标签或 `[本人]`
- AI 推断 → `[AI 推测]`
- AI 下结论 → `[AI 结论]`
- 带风险等级或概率的判断 → `[AI 关键标注]`
- AI 自己行为的反思 → `[AI 元记录]`

用户确认或反驳 AI 推测时，**用追加，不覆盖原文**。例子：

```markdown
[AI 推测 2026-05-10] 这个客户可能更看重交付速度而非价格。
[本人 反馈 2026-05-15] 错了，对方明确说价格优先。
```

不要把第二行覆盖第一行。两行都要在。

## 七、不要把阶段判断写成永久结论

阶段判断只在某个时间窗口成立。

- `03_定格/状态画像_2026-05.md` 里说"用户当前主线是 X" → 5 月有效。
- 6 月情况变了 → 新建 `状态画像_2026-06.md`，不要回去改 5 月的文件。

`02_规律/` 里的概念和模式可以追加修正，但要用 `[修正 YYYY-MM-DD]` 写在原段下方，不重写原段。

## 八、对话结束时做什么

每次对话结束前完成下面这套动作：

1. 更新 `我的当前状态.md § 最近动向`：今天发生了什么、结构意义、需要继续观察什么。
2. 如果有问题被回答了：从 `04_待解/开放问题.md` 移除条目，把答案归档到 `01_事实/` 或对应概念文件。
3. 如果炼出新概念或模式：新建 `02_规律/概念_*.md` 或 `模式_*.md`。
4. 如果出现重大状态变化：新建快照到 `03_定格/`（不覆盖旧的）。
5. 如果有"下次对话用户可能说 X"的预期：写到 `我的当前状态.md § 下次对话推荐入口`。

## 九、隐私模式下的代号纪律

如果项目开启了隐私模式：

1. 文件名、对话里、写文件时，**默认用代号**。
2. 真名只在 `01_事实/相关方笔记/代号映射.md`。
3. 这个映射文件不分享、不截图、不提交 git。
4. 用户若不小心在对话里说了真名，AI 在落盘时自动转换为代号，并在状态文件提示一句"代号映射已更新"。
5. 截图分享时，只截 `00_使用说明.md` 或脱敏文档，不截 `我的当前状态.md` 或 `04_待解/`。

## 十、AGENTS.md 项目级模板

新建项目时，AGENTS.md 写入下面这套（占位替换为实际值）：

```markdown
# AGENTS.md

## 项目定位

这是一个 {领域名} 的决策跟踪项目。使用 `dbs-decision` 四层结构。

## 进入对话先读

1. [00_使用说明.md](./00_使用说明.md)
2. [我的当前状态.md](./我的当前状态.md)
3. 本文件

## 信息权威

本地 Markdown 是唯一权威。AI 记忆是临时缓存。冲突时本地优先。

## 四层规则

| 层 | 规则 |
|---|---|
| `01_事实/` | 只追加 |
| `02_规律/` | 缓慢追加修正，原段不重写 |
| `03_定格/` | 写完不改 |
| `04_待解/` | 完成即清 |

## 来源标签

`[本人]`（默认） / `[AI 推测]` / `[AI 结论]` / `[AI 关键标注]` / `[AI 元记录]` / `[AI 暂定概念 YYYY-MM-DD]` / `[AI 暂定模式 YYYY-MM-DD]` / `[结果回填 YYYY-MM-DD]` / `[本人 反馈 YYYY-MM-DD]` / `[修正 YYYY-MM-DD]`。

AI 标签不能省。追加不覆盖。事实和判断分开。

## 工作原则

- 用户一次性给多条信息 = 综合画像模式，不是被访谈模式。
- 用户进入"问什么办 → 拒方案 → 问新的"循环时，停止给方案，写 [AI 元记录]。
- 不把状态值（"我想 X"）当作承诺契约。
- 不把阶段判断写成永久结论。
- 新增内容默认放素材层（01_事实/），等沉淀后再考虑提炼。

## 对话结束时

更新 `我的当前状态.md § 最近动向`。完成的问题归档。新概念建文件。重大变化新建快照。

## 隐私（如适用）

文件名只用代号。真名只在 `01_事实/相关方笔记/代号映射.md`。这个文件不分享、不提交。
```
